// origin
Welcome to LeetMate

LeetMate is made by a student for students. When I was solving LeetCode problems, they were really challenging and I was losing motivation — but then I thought to check on my classmates to see their progress and to check "How's the Competition." There was no such option or feature available, and at that time I was also learning web development, so I decided to build one. Now by using LeetMate, all students can see the competitive environment and be motivated to catch up with their friends and be at the top of the class.

// the journey
Building LeetMate
21 days to build
30+ bugs fixed
9 API routes

It took me 21 days to build LeetMate. Though it could be built within 5–10 days, it was built while learning the concepts — so it took more time. I learned a lot of things and made many mistakes while working on it. At the end, it was all worth it. LeetMate helped me learn authentication, REST APIs, JWT, databases, backend development, deployment, and working with real users.

"The major learning was — you can make mistakes even in the skill you are most confident in. For me, that was the frontend part."

I used Claude AI to learn concepts and catch my mistakes, but I majorly gained knowledge from official documentation. I will strongly suggest learning from docs rather than YouTube tutorials — it will take time to read and understand, but it is all worth it.

// under the hood
Tech Stack
FastAPI REST APIs Authentication PostgreSQL (Supabase) JWT Tokens HTML CSS JavaScript Render (Backend) Vercel (Frontend)
// what it does
Features
  • 📊Track yours and your friends' live LeetCode stats
  • 🏆Compete on your college leaderboard, filterable by branch
  • 👥Add friends and compare Easy / Medium / Hard solves
  • 📈See exactly how many problems you need to overtake someone
  • 🟢Check who's active and grinding in real time
  • 💬Share feedback directly inside the app
  • 🔄Stats auto-refresh every 30 minutes in the background

This site is built completely from scratch by a fellow CSE student — AI was ONLY used to learn the concepts and to catch my mistakes and to design the website. Bugs may exist, and that's part of the journey. If you find one, use the feedback button inside the app.
